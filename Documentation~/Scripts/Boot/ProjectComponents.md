# ProjectComponents.cs

This is an internal component manager responsible for storing models, managing subscriptions to component addition/removal events, and iterating over active connections.

It stores all active models in the `all` dictionary, using the scene identifier `ProjectContext.scene.key` as the key.
Subscriber management is implemented via the `_listeners` dictionary, which uses string keys generated by the `Key` method from the model and component type names.
When a component is added or removed via the `Add` and `Remove` methods, the class automatically invokes the corresponding callbacks for all registered listeners via the `IComponentListeners` interface.
Method overloads with the `UnloadPool` parameter are used for safe management of subscription lifetimes.
The `ForEach` methods allow you to iterate over existing `ComponentLink<T>` links either globally or within a specific context.
